Hypothesis/Central Question: Given a social contact graph in which vertices carry feature information and edges carry costs derived from endpoint feature similarity, what is the minimum-cost set of edges whose removal keeps steady-state epidemic prevalence below a fixed threshold?

Abstract: This project will explore cost-constrained epidemic containment on real social networks using the Facebook100 dataset. We will study which social connections (edges) to sever, and at what cost, to prevent epidemic spread. Edge costs are derived from node feature similarity via a sigmoid dot-product proxy, reflecting the intuition that severing close social ties is more disruptive than cutting weak ones. We will compare several edge-removal policies, evaluated against a steady-state prevalence threshold under a stochastic SIS epidemic model. 


Technical Breakdown: 
* Cost Proxy: Sigmoid of the dot product of endpoint node feature vectors — σ(⟨x_u, x_v⟩). Nodes with similar features will therefore get high-cost edges (socially close = expensive to cut).
* Epidemic Model:  initially SIR, but focus is on SIS (Susceptible-Infected-Susceptible)
   * Each node in the contact graph occupies one of two states — Susceptible (S) or Infected (I). Infected nodes spread the disease to susceptible neighbors with probability β per time step, and recover back to susceptible (not immune) with probability γ. This cycle repeats indefinitely, allowing reinfection.
   * Epidemic Threshold: τ_c = 1/λ(A), where λ(A) is the spectral radius of the adjacency matrix. If β/γ exceeds this threshold the disease becomes endemic
* Containment Policies/Methods to compare: 
   * Random edge removal — trivial baseline
   * Edge betweenness centrality — remove structural bottleneck edges
   * GNN-based learned policy — GNN learns node embeddings; edge costs derived from embedding similarity; policy selects lowest-cost edges that achieve containment
   * Distance Threshold — within X distance = remove
* Robustness
   * Policy adherence - only X% would listen
   * Real world practicality?
